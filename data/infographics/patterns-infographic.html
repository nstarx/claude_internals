Pattern Selection Flow
Choose the right pattern based on your current task

CLAUDE.md Pattern

  • Project-level context persistence
  • Survives session boundaries
  • Auto-loaded on project open
  • Team knowledge sharing

Layered Context Pattern

  • Stable base + dynamic additions
  • Minimize redundant loading
  • Task-specific overlays
  • Clear separation of concerns

Just-in-Time Pattern

  • Load context when needed
  • Unload when done
  • Keeps window lean
  • Tool-based retrieval

Compression Pattern

  • Summarize lengthy context
  • Extract key information
  • Maintain semantic density
  • 5x+ token savings

When to Use Each Pattern

  • CLAUDE.md: Project conventions, team standards, recurring decisions
  • Layered: Complex projects with multiple subsystems
  • Just-in-Time: Exploratory work, large codebases
  • Compression: Long conversations, meeting notes, documentation
Pro Tip: Combine patterns for best results
CLAUDE.md + Layered + Just-in-Time = Optimal context management
