Things to Avoid

Critical anti-patterns that derail your workflow

Context

  • Reading entire codebases
  • Repeated file reads
  • Ignoring token warnings

Tasks

  • Vague requests
  • No TODO for complex tasks
  • Accepting claims blindly

Code

  • Skipping tests
  • Missing edge cases
  • Monolithic files

Git

  • Working on main
  • Vague commit messages
  • Committing secrets

Security

  • Ignoring validation
  • Hardcoding secrets
  • Disabling security

Tools

  • Bash for file ops
  • Sequential when parallel
  • Ignoring MCP servers

Top 10 Rules

1 Be surgical with file reads
2 Verify completion claims
3 Use TODO for complex tasks
4 Debug test failures
5 Test before commit
6 Reinforce constraints
7 Investigate errors
8 Split monoliths
9 Protect secrets
10 Use feature branches
