Common Antipatterns

Five patterns to avoid for effective context management

1

Over-Loading Context

Problem

Loading entire codebase "just to be safe"

Impact

Token exhaustion, slow performance

Lazy loading with just-in-time retrieval
2

Session Amnesia

Problem

Rediscovering same patterns repeatedly

Impact

Wasted tokens, slower development

Write discoveries to memory immediately
3

Ignoring Memory

Problem

Treating each session as fresh start

Impact

Linear scaling of context cost

/sc:load and /sc:save lifecycle
4

Premature Abstraction

Problem

Complex memory structures before understanding needs

Impact

Maintenance burden, low utilization

Start simple, refine based on usage
5

Stale Memory

Problem

Outdated information in persistent memory

Impact

Incorrect decisions from obsolete context

Memory review cycles, timestamp info

Prevention Checklist

Load only what you need
Persist valuable discoveries
Use session lifecycle commands
Start simple, evolve structure
Review and clean memories
