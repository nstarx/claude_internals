Claude Code: Best User Methodology

Master Claude Code through practical workflows, proven patterns, and strategic approaches to AI-assisted development

What is Claude Code?

Claude Code is Anthropic's official CLI tool that integrates Claude AI directly into your development workflow. It's an interactive coding assistant with a rich set of development tools, persistent sessions, and specialized agents.

Key Advantage: Context-aware development with 200K token working memory, tool integration, and natural language interaction.

Understanding the Foundation

The Context Window: Your Working Memory

The context window is Claude's "RAM" - the total text Claude can process at once. Think of it as your collaboration workspace.

Context Budget
Available: ~200,000 tokens ≈ 150,000 words ≈ 600 pages
Token Conversion:
  • 1 token ≈ 4 characters
  • 1 token ≈ 0.75 words
  • Medium file (300 lines) ≈ 3,000-5,000 tokens

Context Consumption Breakdown

Item Typical Size Impact
Conversation History Variable Accumulates
File Reads 3K-10K per file Persists
Tool Results 1K-5K per result Temporary
System Instructions 10K-30K Pre-loaded

Context Usage Guidelines

  • 0-75% - Normal operation, full capabilities
  • 75-85% - Be selective with file reads
  • 85%+ - Critical, focus on essentials only

Core Methodology: The 4-Phase Workflow

1

Discovery Phase

Goal: Understand the codebase with minimal context usage

Efficient Discovery Pattern
✅ Strategic Approach:
1. Glob "**/*auth*"           → Find relevant files (low cost)
2. Grep "jwt.*login"          → Search for patterns (medium cost)
3. Read auth.js               → Read specific file (high cost)

❌ Inefficient Approach:
1. Read all files in src/     → Burns 50K+ tokens
2. Search manually            → Wastes context
Pro Tip: Use Task tool with Explore agent for complex discovery - it runs in a separate context window.
2

Planning Phase

Goal: Create actionable roadmap before execution

Planning Criteria
Use TODO lists when:
✅ Task has 3+ distinct steps
✅ Multiple files need modification
✅ Task is non-trivial or complex
✅ User provides multiple requirements

Skip TODO lists when:
❌ Single, straightforward operation
❌ Trivial task (< 3 simple steps)
❌ Purely conversational query
Best Practice: Break complex tasks into parallel and sequential operations during planning.
3

Execution Phase

Goal: Make changes efficiently with parallel operations

Parallel vs Sequential
✅ Parallel (Independent Operations):
Read file1.js + Read file2.js + Read file3.js
↓ All complete simultaneously

❌ Sequential (When Not Needed):
Read file1.js → wait → Read file2.js → wait
↓ 3x slower

✅ Sequential (When Required):
Read file.js → Edit file.js → Bash "npm test"
↓ Each depends on previous
⚠️ Critical Rule: Always Read a file before using Edit or Write on existing files.
4

Verification Phase

Goal: Validate changes before committing

Verification Checklist
Before committing:
☑ Run all tests: npm test / pytest
☑ Check build: npm run build
☑ Review diff: git diff
☑ Verify files: git status
☑ Edge cases tested
☑ No console.log() left behind
Trust but Verify: Always ask Claude to show git diff and test results before accepting "completed" claims.

Strategic Patterns for Success

Pattern 1: Symbol-Based Reading (Serena MCP)

Instead of Reading Entire Files
❌ Traditional Approach:
Read user.controller.js      → 5,000 tokens (entire file)
Read auth.service.js         → 8,000 tokens (entire file)
Read database.js             → 10,000 tokens (entire file)
Total: 23,000 tokens

✅ Symbol-Based Approach:
find_symbol "UserController"                → 500 tokens (overview)
find_symbol "UserController/login" depth=1  → 300 tokens (method only)
get_symbols_overview auth.service.js        → 200 tokens (structure)
Total: 1,000 tokens

Savings: 95.7% reduction

Pattern 2: Parallel File Operations

Maximize Efficiency with Parallel Calls
✅ Single Message - Multiple Tools:
Read auth.js + Read user.js + Read config.js + Bash "git status"
↓ All execute simultaneously
Result: 4x faster than sequential

Common Parallel Patterns:
• Read multiple unrelated files
• Multiple independent Grep searches
• Parallel Glob patterns
• Independent Bash commands

Pattern 3: Agent Delegation

When to Delegate to Agents
Use Explore Agent when:
✅ "Where is feature X implemented?"
✅ "How does the auth system work?"
✅ "Find all files related to payments"
✅ Open-ended codebase exploration

Use Plan Agent when:
✅ Complex multi-step implementations
✅ Architecture decisions needed
✅ Multiple valid approaches exist

Use General-Purpose Agent when:
✅ Research tasks requiring iteration
✅ Complex searches with multiple attempts
✅ Multi-step investigations

Power Move: Parallel Agent Execution

Launch multiple agents in ONE message - they run simultaneously with separate 200K context windows:

Task(Explore: auth system) + Task(Explore: database layer) + Task(Explore: API routes)
↓ All complete in parallel
↓ Each has full 200K context
↓ No impact on main session context

Critical Do's and Don'ts

Do's

Be Specific

✅ "Add error handling to login function in auth.js line 45"

❌ "Make the code better"

Start Small, Scale Smart

✅ Target specific files → Make changes → Expand scope

❌ Read entire codebase → Get overwhelmed

Use Feature Branches

✅ Create branch → Work → Test → PR → Merge

❌ Work directly on main/master

Leverage Parallel Operations

✅ Read 5 files simultaneously in one message

❌ Sequential reads, waiting for each

Monitor Context Usage

✅ Check token warnings, adjust at 75%

❌ Ignore warnings until 95%+

Verify Completion Claims

✅ "Show me git diff and list file paths"

❌ "Great, thanks!" (without verification)

Don'ts

Don't Read Everything

❌ "Read all files in src/ so you understand"

Impact: Wastes 50-100K tokens, leaves no room for work

Don't Skip Planning

❌ Starting complex multi-step tasks without TODO list

Impact: Forgotten subtasks, incomplete work

Don't Ignore Test Failures

❌ "Tests failing, just comment them out"

Impact: Hidden broken functionality

Don't Use Bash for File Operations

❌ Bash "cat file.js" or Bash "echo 'code' > file.js"

Use Read/Write tools instead - faster and more reliable

Don't Accept Vague Requests

❌ "Fix all bugs" / "Optimize everything"

Clarify scope and success criteria first

Don't Commit Without Testing

❌ Edit files → git commit → push

Always run tests before committing

Advanced Techniques

The Warmup Effect

Insight: Claude gets progressively better at working with your codebase as the conversation continues. The first few interactions build a mental model.

Pro Tip: Start complex sessions with: "Let me explore the authentication system first" before diving into specific changes.

Context Anchoring

Insight: Early messages have disproportionate influence on Claude's behavior throughout the session.

Pro Tip: State critical constraints in the FIRST message for maximum effect:

Message 1: "Key constraints: React hooks only, no class components,
            TypeScript strict mode, all functions must have tests"
↓ Deeply embedded throughout session

Making Claude Think Harder

Magic Phrases for Deeper Analysis
Level 1 (Basic):
"Add error handling"
↓ Basic try-catch

Level 2 (Better):
"Add comprehensive error handling with logging and recovery"
↓ More careful consideration

Level 3 (Best):
"Add error handling. Think step-by-step about:
1. What could fail?
2. How should we recover?
3. What context should we log?
4. How do we prevent cascading failures?"
↓ Deep analytical mode activated

Other Magic Phrases:
• "Think step-by-step"
• "Explain your reasoning"
• "What could go wrong?"
• "Consider edge cases"
• "Walk me through your approach"

When to Start Fresh vs Continue

Start New Session Continue Current Session
Context usage > 85% Context usage < 75%
Working for > 1 hour Related, interconnected tasks
Switching to different codebase area Good understanding of current area
Claude making "stupid" errors Sequential tasks building on each other
Forgetting earlier constraints Active debugging of same issue
Conversation feels "muddled" Clear progress being made
Session Persistence: Use /sc:save before starting fresh, then /sc:load in new session to restore context.

Claude: I'll help you add JWT authentication. Let me first understand your current setup.

  2. Create JWT middleware in middleware/auth.js
  3. Add login/register endpoints
  4. Protect existing API routes
  5. Add tests for auth flow

[Phase 3: Execution - Mark first item in_progress]
Bash "npm install jsonwebtoken bcrypt"

[Complete item 1, start item 2]

[Complete item 2, start item 3]
Read server.js             → Understand current structure

[Complete item 3, start item 4]
Read routes/api.js

[Complete item 4, start item 5]

[Phase 4: Verification]

[Complete item 5]
✅ JWT authentication implemented:
   • Login endpoint: POST /api/auth/login

Quick Reference: Common Commands

Slash Commands

  • /sc:test - Execute tests with coverage
  • /sc:build - Build and optimize project
  • /sc:git - Intelligent git operations
  • /sc:analyze - Code quality analysis
  • /sc:implement - Feature implementation workflow
  • /sc:troubleshoot - Diagnostic and debugging
  • /sc:save - Save session context
  • /sc:load - Restore session context
  • /sc:help - List all commands

File Operations

  • Glob "pattern" - Find files by pattern
  • Grep "pattern" - Search file contents
  • Read file.js - View file contents
  • Edit file.js - Make precise edits
  • Write file.js - Create new file

Git Operations

  • git status - Check repository status
  • git diff - View changes
  • git log -n 5 - Recent commits
  • Ask Claude to create commits with proper messages
  • Ask Claude to create PRs with gh CLI

Testing & Building

  • npm test / pytest
  • npm run build
  • npm run lint
  • npm install package

Key Takeaways

1

Be Strategic with Context

Use searches and symbol operations before reading entire files. Monitor token usage and adjust at 75%.

2

Plan Before Executing

Use TODO lists for complex tasks (3+ steps). Break down into parallel and sequential operations.

3

Leverage Parallel Operations

Read multiple files, run multiple searches, launch multiple agents - all in one message.

4

Trust but Verify

Always ask for git diff, test results, and file lists. Don't accept completion claims blindly.

5

Use Specialized Agents

Delegate complex exploration, planning, and research to agents - they have separate context windows.

6

Anchor Early

State critical constraints in the first message for maximum influence throughout the session.

7

Start Fresh When Needed

New session at 75-85% context or after 1 hour. Use /sc:save and /sc:load for continuity.

8

Make Claude Think Harder

Use "think step-by-step", "explain your reasoning", and "what could go wrong?" for deeper analysis.

Final Thoughts

Claude Code as a Collaborative Partner

Think of Claude Code as a brilliant junior developer who:

  • Has deep technical knowledge and works incredibly fast
  • Needs explicit requirements and benefits from code review
  • Improves with feedback and gets better throughout the session
  • Works best with guidance, verification, and clear constraints

The more context you provide and the clearer your requirements, the better the results will be.

Remember: Effective use of Claude Code is a skill that improves with practice. Start with these methodologies, experiment with different patterns, and develop your own workflow based on what works best for your projects.
