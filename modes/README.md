# Behavioral Modes

This directory contains mode definitions that modify Claude Code's behavior for specific contexts and workflows.

## Available Modes

- **Brainstorming.md** - Collaborative discovery for vague requirements
- **Business_Panel.md** - Multi-expert business analysis mode
- **Introspection.md** - Meta-cognitive analysis and self-reflection
- **Orchestration.md** - Intelligent tool selection and resource optimization
- **Task_Management.md** - Hierarchical task organization with memory persistence
- **Token_Efficiency.md** - Symbol-enhanced communication for token reduction

## Mode Selection

Modes can be:
- **Auto-activated** by trigger keywords and patterns
- **Manually activated** using flags (e.g., `--brainstorm`, `--introspect`)
- **Contextually selected** based on task complexity and requirements

## Integration

Modes work together with:
- [Core Framework](../core/) - Built on core principles and rules
- [MCP Servers](../mcp/) - Leverage specialized tools
- [Business Features](../business/) - Extended by business panel capabilities
